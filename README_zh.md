# 个人 Wiki 知识管理系统

> 基于 LLM 驱动知识管理理念的知识库系统

---

## 概述

这是一个利用大语言模型（LLM）自动组织和维护的个人知识管理系统。只需添加原始文档，LLM 就会提取知识、建立连接，构建一个不断增长的信息网络。

### 核心特性

- **零维护**：LLM 处理所有组织工作
- **知识复利**：每次添加都会丰富整个知识库
- **双向链接**：每个连接都是相互的，可双向发现
- **来源可追溯**：所有知识都可追溯到原始文档
- **面向未来**：纯 Markdown 文件，无供应商锁定

## 快速开始

### 1. 初始设置

```bash
# 克隆或创建 wiki 目录
git init  # 可选：用于版本控制
```

### 2. 添加第一个文档

```bash
# 将文档放入相应的 raw/ 子目录
# 示例：raw/books/html/巴菲特-1985.html
```

### 3. 使用 LLM 摄入

向你的 LLM 助手提问：
```
"摄入 raw/books/html/巴菲特-1985.html 中的文档"
```

LLM 将：
1. 读取并分析文档
2. 提取实体和概念
3. 创建结构化的 wiki 页面
4. 添加交叉引用
5. 更新索引

### 4. 探索你的 Wiki

- 在 Obsidian 或任何 Markdown 编辑器中打开 `wiki/index.md`
- 通过 `[[链接]]` 在页面间导航
- 向 LLM 提问关于你的知识库

## 系统架构

### 三层设计

```
┌─────────────────────────────────────┐
│     模式层 (CLAUDE.md)              │  ← LLM 指令
├─────────────────────────────────────┤
│     Wiki 层 (wiki/)                 │  ← 知识内容
├─────────────────────────────────────┤
│     原始层 (raw/)                   │  ← 源文档
└─────────────────────────────────────┘
```

- **原始层**：不可变的源文档（论文、文章、笔记、书籍）
- **Wiki 层**：LLM 维护的结构化内容，带交叉引用
- **模式层**：指导 LLM 行为的配置

### 页面类型

1. **实体**（`wiki/entities/`）：人物、组织、项目、工具
2. **概念**（`wiki/concepts/`）：理论、框架、方法、原则
3. **来源**（`wiki/sources/`）：文档摘要和分析
4. **综合**（`wiki/synthesis/`）：综合分析和比较

## 核心操作

### 1. 摄入：添加知识

**目的**：将原始文档转化为结构化知识

**用法**：
```
"摄入 raw/books/pdf/巴菲特-2000.pdf 中的文档"
```

**流程**：
1. LLM 读取源文档
2. 识别实体（人物、组织、工具）
3. 提取概念（理念、方法、框架）
4. 创建来源摘要页面
5. 创建/更新实体和概念页面
6. 添加双向交叉引用
7. 更新 `wiki/index.md`

### 2. 查询：探索知识

**目的**：提问和发现关联

**用法**：
```
"巴菲特关于价值投资的核心观点是什么？"
"经济护城河有哪些类型？"
"1987年股灾期间巴菲特如何应对？"
```

### 3. 检查：维护质量

**目的**：验证 Wiki 健康状况并修复问题

**用法**：
```
"检查 Wiki 健康状况"
"查找孤立页面"
"识别损坏的链接"
```

## 使用指南

### 添加文档

1. **识别文档类型**：信件、文章、书籍等
2. **放入适当目录**：
   - HTML 信件 → `raw/books/html/`
   - PDF 信件 → `raw/books/pdf/`
3. **使用 LLM 摄入**："摄入 [文件名]"

### 浏览知识

#### 使用 Obsidian（推荐）

1. 将此目录作为 Vault 在 Obsidian 中打开
2. 导航到 `wiki/index.md`
3. 点击 `[[链接]]` 探索关联页面
4. 使用图形视图可视化连接

#### 使用任意 Markdown 编辑器

1. 在编辑器中打开 `wiki/index.md`
2. 手动跟随链接（搜索链接文本）
3. 使用搜索功能查找主题

### 提问

使用自然语言查询你的知识库：

```
"巴菲特对可口可乐的投资逻辑是什么？"
"解释内在价值的概念"
"哪些信件讨论了保险浮存金？"
"总结 2008 年金融危机的相关内容"
```

## 支持的格式

### 直接支持（LLM 可读取）

- Markdown (`.md`)
- 文本文件 (`.txt`)
- PDF (`.pdf`)
- HTML (`.html`)
- JSON (`.json`)
- CSV (`.csv`)
- 代码文件 (`.py`, `.js`, `.java` 等)

### 需要多模态 LLM

- 图片 (`.png`, `.jpg`)
- 扫描文档

### 需要预处理

- 音频 (`.mp3`, `.wav`) → 先转录
- 视频 (`.mp4`) → 提取音频/字幕

## 设计理念

### 基于 LLM Wiki 原则

本系统实现了 LLM Wiki 理念，将结构化文件系统作为 LLM 可自主维护的知识数据库。

### 核心原则

1. **简洁性**：纯 Markdown 文件，无复杂基础设施
2. **LLM 原生**：从设计之初就为 LLM 维护而构建
3. **双向性**：每个连接都是双向的
4. **可追溯性**：所有知识都链接回源
5. **可扩展性**：为未来增强做好准备
